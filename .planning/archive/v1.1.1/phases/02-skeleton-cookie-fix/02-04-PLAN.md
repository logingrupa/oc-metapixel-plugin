<objective>
Ship the `PixelHead` component: an October CMS component that renders alongside the theme's existing `partials/facebook_pixel.htm` (never replaces it — coexistence is the SKEL-04 contract). The component is the ONE place in Phase 2 where a server-generated `event_id` reaches the browser, which is the cornerstone of the Phase 3+ dedup contract (Meta dedupes on `event_id` match within ±10s; server-side authoritative UUID v4 is the only deterministic source).

Purpose: The existing theme partial fires `fbq('init', pixel_id, {em, fn, ln, ph, external_id})` (PII in init) and `fbq('track', 'PageView')` with NO eventID — both are bugs. The PII-in-init pattern double-sources `external_id` and breaks Phase 3 dedup. The missing eventID means Meta counts every Pixel call as a separate event. PixelHead corrects both: PII-free init + eventID-stamped tracks. Phase 2 fires PageView client-side only; the CAPI twin lands in Phase 4 FUN-01.

Output: `components/PixelHead.php`, `components/pixelhead/default.htm` (twig partial), `Plugin::registerComponents()` declaring the alias `pixelHead`, and `tests/Feature/PixelHeadTest.php` locking the 7 onRun behaviors. Browser-side functional verification deferred to Phase 4 alongside CAPI dispatch.

Directory casing note: this plan uses lowercase `components/` (top-level directory) and lowercase `components/pixelhead/` (template subdirectory). Sibling-plugin precedent: `plugins/logingrupa/storeextender/components/CustomProductPage.php` + `plugins/logingrupa/storeextender/components/lazypromoblockloader/`, `plugins/logingrupa/campaignpricingshopaholic/components/CampaignPricing.php` + `plugins/logingrupa/campaignpricingshopaholic/components/campaignpricing/`. October's `ComponentBase::__construct()` derives `$this->dirName = strtolower(str_replace('\\', '/', $className))` (verified at modules/cms/classes/ComponentBase.php line 95) — the template lookup is always lowercase.
