<objective>
Ship `Logingrupa\Metapixelshopaholic\Classes\Meta\MetaClient` — the single HTTP boundary to Meta's Graph API v20.0 `/events` endpoint. Stateless, single-shot, retry-free (retry happens at the queue-job layer in plan 03-05).

Purpose: Centralize the HTTP boundary so every CAPI dispatch from anywhere in the plugin (Purchase Phase 3, every funnel event Phase 4) flows through one classification gate. Transient-vs-permanent error classification is encoded by throwing the right typed exception (03-02). Pixel ID + access token are read lazily at send-time so missing-config failures are localized to the event boundary, not the boot path (per CONTEXT Area 3 Q4).

Output: One production class + one unit-level test suite with Guzzle MockHandler covering the 7 send-time invariants.
</objective>

    The `'http_errors' => false` is critical — Guzzle's default is to throw `BadResponseException` for 4xx/5xx; we disable that and handle the status code explicitly so transient-vs-permanent classification flows through a single switch on `getStatusCode()`.

    Public method:
    ```
    /**
     * Send a CAPI events payload to Meta Graph API v20.
     *
     * @param  array<string, mixed>  $arPayload  envelope with key "data" => list of event records
     * @return array<string, mixed>  decoded Graph API response
     * @throws MissingPixelConfigException   when pixel_id Setting is empty
     * @throws MissingCapiTokenException     when capi_access_token Setting is empty
     * @throws MetaApiTransientException     on HTTP 408/429/500/502/503/504 or ConnectException
     * @throws MetaApiPermanentException     on any other HTTP error
     */
    public function send(array $arPayload): array
    ```

    `send()` body sequence:
    1. Read `$sPixelId = $this->readSetting('pixel_id');`. If empty → throw `new MissingPixelConfigException('Meta Pixel ID is not configured', ['setting_key' => 'pixel_id']);`.
    2. Read `$sAccessToken = $this->readSetting('capi_access_token');`. If empty → throw `new MissingCapiTokenException('Meta CAPI access token is not configured', ['setting_key' => 'capi_access_token']);`.
    3. Read `$sTestEventCode = $this->readSetting('test_event_code');` (may be empty — that's fine; only added to query when non-empty).
    4. Build query: `$arQuery = ['access_token' => $sAccessToken];` then `if ($sTestEventCode !== '') { $arQuery['test_event_code'] = $sTestEventCode; }`.
    5. Build URI path: `$sUri = $sPixelId . '/events';` (relative to base_uri).
    6. Wrap try/catch:
       ```
       try {
           $obResponse = $this->obClient->request('POST', $sUri, [
               'query' => $arQuery,
               'json'  => $arPayload,
           ]);
       } catch (ConnectException $obException) {
           throw $this->makeTransientException(
               'Meta CAPI network failure: ' . $obException->getMessage(),
               null,
               $obException,
           );
       } catch (RequestException $obException) {
           $iStatus = $obException->hasResponse() ? $obException->getResponse()->getStatusCode() : null;
           throw $this->classifyException($iStatus, $obException);
       }
       ```
    7. After the try/catch (response received), classify by status code:
       ```
       $iStatus = $obResponse->getStatusCode();
       $sBody = (string) $obResponse->getBody();
       if ($iStatus >= 200 && $iStatus < 300) {
           $arDecoded = json_decode($sBody, true);
           return is_array($arDecoded) ? $arDecoded : [];
       }
       if (in_array($iStatus, self::TRANSIENT_STATUS_CODES, true)) {
           throw $this->makeTransientException(
               'Meta CAPI transient error (HTTP ' . $iStatus . '): ' . $sBody,
               $iStatus,
           );
       }
       throw $this->makePermanentException(
           'Meta CAPI permanent error (HTTP ' . $iStatus . '): ' . $sBody,
           $iStatus,
       );
       ```

    Private helper `readSetting(string $sKey): string`:

    $mValue = Settings::get($sKey, '');
