EventPixel::onRun shape:
- Read component properties: subject_class, subject_slug_field, subject_type (operator-supplied alias), event_name, slug (optional — falls back to route param).
- Resolve the subject's row id: `$iSubjectId = (int) $sSubjectClass::query()->where($sSlugField, $sSlug)->value('id');` Skip render if iSubjectId is 0.
- Direct DB::table query for the CAPI row: where(subject_type=$sSubjectType, subject_id=$iSubjectId, event_name=$sEventName, channel='capi') first(event_id, event_time, payload).
- If null → return (no emit).
- Check pixel-side row absent: same where but channel='pixel'; exists() → if true, return (already fired).
- Decode payload (Pitfall 8: DB::table returns longText as raw string; explicit json_decode required).
- Set page var `eventPixelData` with event_id_json + event_name_json + custom_data_json (all JSON-encoded with HEX flags).

EventPixel onMarkFired AJAX:
- Reads Input: subject_type, subject_id, event_name, event_id.
- VALIDATES event_id matches the CAPI row's event_id (un-injectable — browser can only echo back what server emitted).
- INSERT IGNORE the pixel-side row via `DB::table('logingrupa_metapixel_event_log')->insertOrIgnore([...])`. UNIQUE race-fence handles reload double-write.
- Returns `['ok' => true]` on success, `['ok' => false, 'error' => '...']` on failure (validation or DB error).

PixelHead::onRun:
- Flush ThemeEventCollector (returns list<array> of pushed events).
- For each event: emit `<script>fbq("track", <name_json>, <custom_data_json>);</script>` block via Twig partial.
- If `also_dispatch_capi:true` in the event dict, build ThemeActionEvent + PayloadBuilder + SendCapiEvent::dispatch for CAPI mirror.
- Catch any throwable in the CAPI mirror; Log::warning + continue (do NOT break page render).
