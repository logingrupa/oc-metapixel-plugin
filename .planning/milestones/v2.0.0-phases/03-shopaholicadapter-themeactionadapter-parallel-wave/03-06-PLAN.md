Twig dot-notation mount (RESEARCH.md A1 — implementation surface in Plugin::boot):
```
\Cms\Classes\Controller::extend(function ($obController) {
    $obController->bindEvent('page.beforeRenderPage', function () use ($obController) {
        $obController->vars['metapixel'] = App::make(ThemeEventCollector::class);

});
```
Then in Twig: `{% do this.metapixel.pushEvent({...}) %}` — `this` resolves to the controller, `this.metapixel` reads the dynamic `vars['metapixel']` set during beforeRenderPage. The collector exposes a `pushEvent(array $arEvent): void` method that Twig's attribute resolver maps to via property → method resolution.

**Note:** Twig's attribute resolver tries property → method → method('get'.Name) → method('is'.Name) → method('has'.Name) in order. `this.metapixel.pushEvent({...})` resolves to `$obCollector->pushEvent($arEvent)`. The collector also exposes `push` (canonical name) for direct PHP callers; `pushEvent` is the Twig-facing entry point.
