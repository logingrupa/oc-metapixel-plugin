<objective>
Close UAT test 9 (05-UAT-CUTOVER.md Gaps): wire a browser fbq AddToCart that
reuses the server-generated CAPI event_id and carries full custom_data, so Meta
deduplicates the browser+server pair BY EVENT ID instead of the fragile fbp
fallback. Implements deferred decision D-07 (browser AddToCart wire), pulled into
scope pre-launch by the operator for better ad pricing.

`eloquent.created` of CartPosition with a UUIDv4 event_id and a channel='capi'
EventLog row (subject_type='shopaholic.cart_position'). This plan adds a
PIXEL-ONLY browser emitter that READS that already-generated event_id and
custom_data, writes the channel='pixel' twin row, and returns an executable fbq
script — with a hard guard against any second CAPI dispatch.

Purpose: true event_id deduplication for AddToCart (per D-07); eliminates the
fbp-only fallback observed 2026-07-02.
Output: server resolver (reads capi row, writes pixel twin), AJAX boundary
handler, theme follow-up call, and tests.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → Metapixel::onMarkAddToCart | Untrusted offer_id crosses into the AJAX handler |
| stored capi EventLog payload → returned fbq script | Server-authored data echoed into an inline browser script |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05G-01 | Information Disclosure | resolveBrowserPixel offer_id lookup | medium | mitigate | Position resolved via CartProcessor current-session cart scoped by getByCart — offer_id only selects among the caller's own positions; cannot read another cart's event_id |
| T-05G-02 | Tampering (XSS) | returned fbq script | high | mitigate | FbqScriptBuilder JSON_HEX_TAG/QUOT/AMP/APOS encoding on all interpolated values; no raw string concat of untrusted input |
| T-05G-03 | Spoofing | event_id origin | high | mitigate | event_id is read server-side from the capi EventLog row only; browser never supplies it (event_id direction lock server → frontend) |
| T-05G-04 | Denial of Service | onMarkAddToCart endpoint | low | mitigate | Reuses ThemeAjaxHandler per-IP+session RateLimiter (30/60s) |
| T-05G-05 | Repudiation | double CAPI dispatch | high | mitigate | resolveBrowserPixel contains no SendCapiEvent::dispatch; unit test asserts not-dispatched; CartPositionWatcher eloquent.created remains sole CAPI emitter |
</threat_model>
