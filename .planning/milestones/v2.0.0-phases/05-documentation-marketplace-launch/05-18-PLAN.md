<refactor_principles>
Read before starting any task. These govern EVERY change in this plan:

- **Behaviour-preserving ONLY.** Extract / move private methods. No logic changes, no control-flow semantics changes, no output changes. All 561 pest tests MUST stay GREEN. No public method signature changes (`onBeforeRun`, `markAddToCartPixel`, `subscribe`, `handle`, `dispatchForOfferSwitch`, `flushDeferredFromController`, `renderDeferredBlocks` and their parameter/return shapes are frozen).
- **How phpmd class complexity works (critical).** `ExcessiveClassComplexity` is the sum of every method's CyclomaticComplexity (WMC). Extracting a body into a NEW private method in the SAME class does NOT reduce class WMC — it adds a base +1. Class WMC only drops when decision points leave the class entirely (Move Method to a collaborator). This is why Task 1 needs a collaborator class, while Tasks 2 and 3 (which have NO class-level violation) use plain intra-class extraction.
- **CyclomaticComplexity fires at ≥ 10** (onBeforeRun at exactly 10 is flagged). Target every method to CC ≤ 9 for margin. **NPath fires at > 200**; target ≤ 200. **ExcessiveClassComplexity fires at > 50**; target ThemeAjaxHandler class WMC ≤ 48 for margin.
- **No phpmd suppression.** Do NOT edit phpmd.xml thresholds and do NOT add `@SuppressWarnings` annotations. Fix by extraction/move per project standard (Tiger-Style short functions). None of these violations is structurally unavoidable.
- **Project conventions (parent + plugin CLAUDE.md):** Hungarian notation for all new locals/params (`$ob`, `$ar`, `$i`, `$s`, `$b`, `$f`; PHPMD ShortVariable min 4 — no `$ob`-less short names). camelCase method names. Laravel short docblocks (one-line summary + `@param`/`@return`). NO comment-pollution markers (`// refactor`, `// gap`, `// CR-XX`, `// Phase N`). No `assert()`. No `declare(strict_types=1)` requirement. PHP `^8.3 || ^8.4` dual — no 8.4-only syntax (no property hooks, asymmetric visibility, `array_find`/`array_any`/`array_all`/`array_find_key`, `#[\Deprecated]`).
- **phpstan level 10 must stay green.** Preserve every existing type-narrowing guard verbatim when moving code (e.g. `is_string($mKey)` key narrowing, `is_numeric(...) ? (int) ... : 0`, `is_array(...)` checks). Moved code carries its narrowing with it.
- **Scope discipline.** Touch ONLY the four listed files. Do not modify phpstan.neon, phpunit.xml, or any test file unless the coverage gate (`pest --coverage --min=90`) actually drops below 90 after the move — and if it does, add a focused test rather than lowering the gate.
</refactor_principles>
