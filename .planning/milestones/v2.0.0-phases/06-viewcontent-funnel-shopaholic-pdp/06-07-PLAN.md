> ## ViewContent funnel (Shopaholic PDP)
>
> Phase 6 closes the conversion funnel at offer-level grain:
>
> ```
> ViewContent (offer SKU)  ‚Üí  AddToCart (offer SKU)  ‚Üí  Purchase (offer SKU per line item)
> ```
>
> Operators with Lovata.Shopaholic + Lovata.OrdersShopaholic installed get ViewContent firing automatically on PDP render plus on every offer-selector change, with zero theme code.
>
> ### Install
>
> Add the `[productPixel]` component to the same layout/page that hosts Shopaholic's `[ProductPage]` component. Octobercms layout markup example (description, not code-fenced for plan): `[productPixel]` between the page metadata block and `[ProductPage]`.
>
> No properties to configure. The component is a no-op when ProductPage is not active OR when `PluginGuard::isDisabled()` (empty `pixel_id` Setting).
>
> ### How it works
>
> 1. Lovata fires `Event::fire('shopaholic.product.open', [$obProduct])` inside `ProductPage::getElementObject` after active + site guards pass.
> 2. `ProductPageWatcher` (auto-subscribed when Lovata.OrdersShopaholic is installed) builds the ViewContent payload, dispatches `SendCapiEvent` to the queue, and pushes a record onto `ThemeEventCollector`.
> 3. At `cms.page.beforeRenderPage`, PixelHead drains the collector into the deferred-flush buffer. The browser receives an inline `<script>fbq("track", "ViewContent", {...}, {eventID: ...})</script>` with the same event_id the server CAPI dispatch carries ‚Äî Meta dedupes them automatically.
> 4. When the operator's theme changes the `[name="offer_id"]` element on the PDP (any `<select>`, radio, hidden input ‚Äî Shopaholic canonical name), the inline JS in `[productPixel]` posts to `Metapixel::onFireEvent` with `subject_type: 'shopaholic.product'` + `subject_id: <pid>` + `offer_id: <oid>`. The server resolves the alias via `AdapterRegistry::resolveByAlias`, re-validates via `ShopaholicProductAdapter::loadSubject`, dispatches CAPI, and returns a fresh inline script for the browser to fire.
>
> ### Soft-gate behavior
>
> The offer-switch JS soft-gates on `window.__metapixelProduct` ‚Äî a server-injected global. It fires only when ProductPixel rendered (= PDP context). Cart-modal bonus-box `[name="offer_id"]` selectors on non-PDP pages do NOT trigger spurious ViewContent (T-6-06 mitigation).
>
> ### Test events
>
> Set `test_event_code` in Settings ‚Üí the server CAPI payload + the browser fbq both carry it. Verify dedup in Meta Events Manager Test Events panel.
