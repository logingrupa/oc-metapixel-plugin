<objective>
Refactor `PixelHead` so the `ThemeEventCollector` flush happens at `cms.page.beforeRenderPage` instead of inside `onRun`, while preserving the base PageView emission and the per-request action_key shape.

Purpose: page-tier components (notably `Shopaholic\ProductPage` in plan 06-05) push events to `ThemeEventCollector` from their own `onRun`. Today `PixelHead.onRun` runs in the layout-component pass which executes BEFORE page-tier components, so collector pushes arrive too late. Moving the flush to `cms.page.beforeRenderPage` (which fires AFTER every component `onRun`, per RESEARCH §1 line 421) unblocks the entire ViewContent funnel.

Output:
- `components/PixelHead.php` keeps `onRun()` emitting `pixelHeadBase` only; adds `static flushDeferredFromController(CmsController $obController): void` reading from the singleton `ThemeEventCollector` and pushing script blocks (with `eventID` 4th-arg when present) into the singleton `PixelHeadDeferredFlushBuffer`. Class-level PHPDoc gains the timing-contract docblock from RESEARCH §12.
- `classes/helper/PixelHeadDeferredFlushBuffer.php` is a new request-scoped singleton with `setBlocks(list<string>): void`, `getBlocks(): list<string>`, `clear(): void`.
- `components/pixelhead/default.htm` renders deferred blocks from the buffer via a static `PixelHead::renderDeferredBlocks(): string` markup helper invoked from Twig with `{{ renderDeferredBlocks()|raw }}` — does NOT use `$this->page['pixelHeadBlocks']` for deferred content.
- `Plugin.php` registers `PixelHeadDeferredFlushBuffer` as a singleton in `register()`; adds a SECOND `cms.page.beforeRenderPage` listener (sibling to the existing ThisVariable injector at line 83-89) invoking the new static.
- The 4 PixelHeadDeferredFlushTest stubs from plan 06-01 turn GREEN.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Page-tier `onRun` push → cms.page.beforeRenderPage flush | Untrusted in principle (third-party plugins can push); collector deserializes only structured array per `ThemeEventCollector::push()` guard. |
| `cms.page.beforeRenderPage` listener → Twig render | The listener writes into the `PixelHeadDeferredFlushBuffer` singleton; default.htm reads via `renderDeferredBlocks()` markup helper. Bypassing the buffer is impossible from Twig. |

|-----------|----------|-----------|-------------|-----------------|
