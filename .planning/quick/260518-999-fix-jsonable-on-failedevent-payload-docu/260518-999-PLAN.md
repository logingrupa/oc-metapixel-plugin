- `protected $fillable` — mass-assignable attributes

    1. **Override of parent Hungarian rule for October model property names.** Parent `/home/forge/nailscosmetics.lv/CLAUDE.md` mandates Hungarian (`$ar`, `$ob`, `$b`, ...) for ALL variables. This plugin's October models use the Laravel-standard property names October itself ships: `$table`, `$fillable`, `$jsonable`, `$casts`, `$rules`, `$customMessages`, `$attributeNames`, plus October relationship arrays (`$hasOne`, `$hasMany`, `$belongsTo`, `$belongsToMany`, `$morphTo`, `$morphOne`, `$morphMany`, `$morphToMany`, `$morphedByMany`, `$attachOne`, `$attachMany`). Rationale: october/boost rules supersede Hungarian for October-defined property names — October's docblock examples, scaffolding (`create:model`), and YAML config generators all assume Laravel-standard names; renaming them to `$arFillable` breaks IDE autocompletion, breaks PHPStan's larastan inheritance resolution at level 10 for the Validation trait, and silently breaks any third-party plugin reading the model via reflection. Local variables and methods (`$obSubject`, `$arPayload`, `getEventName()`) stay Hungarian per parent rule.

    2. **JSON columns use `$jsonable`, not `'array'` cast.** October's idiomatic JSON-in-text pattern is `protected $jsonable = ['column']` when the migration column is `text` or `longText`. The Eloquent `'array'` cast is functionally equivalent for round-trip but bypasses October's HasJsonable trait and reads as a Laravel-Eloquent idiom rather than an October idiom. Reference: `models/FailedEvent.php $jsonable = ['payload']` over `updates/CreateMetapixelFailedEventsTable.php $obTable->longText('payload')`. Scalar casts (`int`, `bool`, `datetime`) continue to use `$casts`.

    3. **Validation trait — skip for internal log models.** Internal append-only log / dead-letter models (`EventLog`, `FailedEvent`) do NOT use `October\Rain\Database\Traits\Validation` and define no `$rules` array. Rationale: both tables are written exclusively by server-side helpers (`EventLogWriter::record`, `SendCapiEvent::writeFailedEvent`) which validate input at the helper boundary (Tiger-Style fail-fast — see EventLogWriter's `subject_id > 0` guard, MetaClient's `MissingPixelConfigException` / `MissingCapiTokenException`). There is no user-input boundary that could deliver a malformed row. Settings (the one user-input model in this plugin) extends Lovata.Toolbox CommonSettings which carries its own RainLab.Translate-aware validation pipeline. Models with a user-input boundary that may land in v2.1+ (e.g. an admin-facing operator-supplied row) MUST add the Validation trait + `$rules`.
