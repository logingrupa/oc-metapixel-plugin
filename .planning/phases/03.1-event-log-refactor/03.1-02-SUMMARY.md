### `Logingrupa\Metapixelshopaholic\Classes\Helper\SiteResolver`

```php
public static function getActiveSiteId(): ?int;
```

- Returns `int` when October 4 SiteManager is bound and `Config::get('system.active_site')` is numeric.
- Returns `null` when SiteManager class missing, active site unset, or any Throwable during probe.
- Callers MUST pass the return directly to `EventLogWriter::record` — DO NOT cast `null` to 0 (UNIQUE NULL-distinct semantics rely on the actual NULL).

### `Logingrupa\Metapixelshopaholic\Classes\Helper\EventLogWriter`

```php
public static function record(
    string $sEventId,
    string $sEventName,
    string $sChannel,
    object $obSubject,
    ?string $sSecretKey,
    int $iEventTime,
): bool;
```

- **Returns `true`** — this call's `insertOrIgnore` actually inserted a row (race winner). Caller MAY proceed with HTTP POST / fbq fire.
- **Returns `false`** — UNIQUE blocked the INSERT (race loser), OR the subject had no numeric `getKey()` (logged warning), OR a Throwable was caught (logged critical, fail-safe). Caller MUST NOT proceed with HTTP POST / fbq fire.

## Decisions Made

- **5 tests, not 4 — SiteResolver CLI null folded into EventLogTest.** BRIEF REFAC-04 line 142 specifies "One unit test asserts null in CLI context"; plan output spec line 428 explicitly allows folding that test into EventLogTest OR creating a separate SiteResolverTest, "either is acceptable." Chosen approach: fold-in. Same hermetic harness, same `setUp`, no duplicated boot — net one fewer file with zero coverage loss for the SiteResolver null path.
- **`(new EventLog())->table` over hardcoded literal.** EventLogWriter reads the table name through the model's `$table` property at the INSERT call site. PATTERNS.md lines 541-542 flag this as the canonical DRY approach. If a future plan renames the table, the model is the single source of truth — writer follows automatically.
- **`extractSubjectId` is private, not a separate trait.** The subject-id narrowing pattern is identical to `OrderStatusWatcher::intOrZero` (Phase 3 PHPSTAN-01). Promoting it to a trait or shared helper is premature — there are only two call sites of this shape in the plugin (this writer + Wave 3 watcher) and the duplication is two lines, not twenty. Keep YAGNI.
- **Hungarian PHPDoc only on method signatures + local variables, NOT on column names.** Model column names stay snake_case (`event_id`, `event_name`, `subject_type`, etc.) per Eloquent convention. The Hungarian convention applies to METHOD signatures (`$sEventId`, `$obSubject`, `$iEventTime`) and local variables (`$iAffected`, `$mKey`, `$obException`). Mirrors FailedEvent.php precedent.

## Threat Surface — No New Flags
