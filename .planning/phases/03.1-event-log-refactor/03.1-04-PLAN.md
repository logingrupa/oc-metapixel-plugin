## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → onMarkFired AJAX | Anyone can POST to the AJAX endpoint with any event_id. Server-authoritative validation is the security boundary. |
| Twig partial → fbq | The IIFE runs in the browser. Pixel id + custom_data already validated server-side via PH-01 + PH-02. |
| Component → EventLog table | All reads must be SiteResolver-scoped to prevent one site's CAPI row from suppressing another site's Pixel render. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3.1-18 | Spoofing | onMarkFired AJAX | mitigate | submitted event_id must match the server's CAPI row event_id. Mismatch → reject + log warning. UUIDv4 has ~122 bits entropy — guessing is computationally infeasible. |
| T-3.1-19 | Tampering | onMarkFired payload | mitigate | EventLogWriter::record uses the SERVER's event_time from the CAPI row, NOT a client-supplied value. Client cannot manipulate the persisted event_time. |
| T-3.1-20 | Repudiation | Pixel-fire claim | mitigate | EventLog row records `fired_at` + `created_at` server timestamps + event_id. Audit trail is append-only (UNIQUE constraint blocks updates). |
| T-3.1-21 | Information Disclosure | onMarkFired Log::warning on mismatch | mitigate | Log captures `submitted_event_id_len` only (an integer length), NEVER the submitted_event_id value itself — prevents log-injection by attacker-controlled UUID string. Order id is integer, no PII. |
| T-3.1-22 | Denial of Service | onMarkFired flood | accept | Lovata.OrdersShopaholic already rate-limits /checkout/{slug}. EventLogWriter is INSERT-only; UNIQUE constraint blocks duplicate work. October's Larajax handler is gated by /checkout/{slug} page resolution. |
