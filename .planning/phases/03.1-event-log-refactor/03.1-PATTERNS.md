```php
class DropMetaPurchaseColumnsFromOrdersTable extends Migration
{
    const TABLE_NAME = 'lovata_orders_shopaholic_orders';
    const COLUMN_ID = 'meta_purchase_event_id';
    const COLUMN_TIME = 'meta_purchase_event_time';
    const INDEX_NAME = 'lovata_orders_shopaholic_orders_meta_purchase_event_id_index';

    public function up(): void
    {
        if (!Schema::hasTable(self::TABLE_NAME)) {
            return;
        }

        if (!Schema::hasColumn(self::TABLE_NAME, self::COLUMN_ID)
            && !Schema::hasColumn(self::TABLE_NAME, self::COLUMN_TIME)) {
            return;
        }

        Schema::table(self::TABLE_NAME, function (Blueprint $obTable): void {
            if (Schema::hasColumn(self::TABLE_NAME, self::COLUMN_ID)) {
                $obTable->dropIndex(self::INDEX_NAME);   // MIG-02: drop index BEFORE column on SQLite
            }
            $obTable->dropColumn([self::COLUMN_ID, self::COLUMN_TIME]);
        });
    }

    public function down(): void
    {
        // Re-add the columns + index for downgrade safety. Matches the
        // legacy migration's `up()` byte-for-byte (lines 58-66 of
        // add_meta_purchase_event_id_to_orders_table.php).
        Schema::table(self::TABLE_NAME, function (Blueprint $obTable): void {
            if (!Schema::hasColumn(self::TABLE_NAME, self::COLUMN_ID)) {
                $obTable->string(self::COLUMN_ID, 36)->nullable()->after('secret_key')->index();
            }
            if (!Schema::hasColumn(self::TABLE_NAME, self::COLUMN_TIME)) {
                $obTable->unsignedBigInteger(self::COLUMN_TIME)->nullable()->after(self::COLUMN_ID);
            }
        });
    }
}
```

**Bug-watch (MIG-02):** SQLite cannot drop indexed columns. `dropIndex` MUST precede `dropColumn` in the same closure (verified in existing file lines 84-91). The hermetic SQLite test harness applies this migration via `MetapixelTestCase::bootOrdersTable() + this migration`.

### `classes/helper/SiteResolver.php` (NEW helper — October 4 multi-site SDK probe)

**Full file:**
```php
<?php

declare(strict_types=1);

namespace Logingrupa\Metapixelshopaholic\Classes\Helper;

use Illuminate\Support\Facades\Log;
use System\Classes\SiteManager;
use Throwable;

/**
 * Phase 3.1 multi-site site_id resolver.
 *
 * Reads the active site_id from October 4's SiteManager singleton. Returns null
 * in three cases:
 *   1. SiteManager class not registered (single-site installation — plugin
 *      shipped to operator who didn't install the multi-site module).
 *   2. SiteManager::instance()->getActiveSiteId() returns null (no active site
 *      bound — CLI/queue context outside a request lifecycle).
 *   3. Any Throwable during SDK probe (defensive — Tiger-Style boundary catch
 *      mirrors PluginGuard.php Settings-read defensive boundary).
 *
 * UNIQUE constraint on event_log treats NULL site_id as a distinct value, so
 * single-site installs (returns null) and multi-site installs (returns int)
 * coexist correctly in the same table.
 *
 * Stateless — no Singleton trait, no container binding, no memo. Each call is
 * a cheap Config::get('system.active_site') read.
 */
final class SiteResolver
{
    /**
     * Return the active site_id from October 4 SDK, or null when absent/disabled.
     *
     * Returns null in CLI / queue / single-site contexts. Callers MUST treat
     * null as "no site scoping" — the UNIQUE constraint will still de-dupe
     * because NULL is a distinct value under MySQL UNIQUE semantics.
     */
    public static function getActiveSiteId(): ?int
    {
        try {
            if (!class_exists(SiteManager::class)) {
                return null;
            }

            $mId = SiteManager::instance()->getActiveSiteId();

            if ($mId === null) {
                return null;
            }

            return is_numeric($mId) ? (int) $mId : null;
        } catch (Throwable $obException) {
            // silent: SDK probe failure must NOT cascade — Tiger-Style
            // boundary catch. Log warning so operator sees infra drift.
            Log::warning('Metapixel: SiteResolver SDK probe failed — treating as single-site', [
                'meta_pixel.exception' => get_class($obException),
                'meta_pixel.message' => $obException->getMessage(),
            ]);

            return null;
        }
    }
}
```

```php
/**
 * Phase 3.1 race-fence writer.
 *
 * Inserts a row into `logingrupa_metapixel_event_log` using `insertOrIgnore`
 * so the database UNIQUE constraint
 * `(subject_type, subject_id, event_name, channel, site_id)` is the atomic
 * race fence. Concurrent INSERTs collide on the constraint; exactly one wins.
 *
 * Returns:
 *   - true  — this caller's INSERT created the row (race winner — proceed
 *             with HTTP POST to Meta or with browser fbq fire).
 *   - false — UNIQUE caught it (race loser OR retry of a successful previous
 *             attempt — no-op gracefully).
 *
 * Used by:
 *   - SendCapiEvent::handle()    — channel='capi', BEFORE MetaClient::send.
 *   - PurchasePixel::onMarkFired — channel='pixel', AFTER browser fbq call.
 *
 * Defensive Tiger-Style boundary catch on Throwable: DB-write failure during
 * record() does NOT cascade. Returns false on DB error (treats infra failure
 * as "lost race" — fail-safe: peer assumed to have won, no double-fire).
 * Mirrors SendCapiEvent::writeFailedEvent silent-catch precedent (T-03-22).
 */
final class EventLogWriter
{
    /**
     * Atomically record an event_log row, leveraging the UNIQUE constraint as
     * race fence. Returns true on race win, false on race loss / DB error.
     *
     * @param string $sEventId    UUIDv4 paired between CAPI + Pixel for Meta dedup.
     * @param string $sEventName  'Purchase' (3.1) / Phase 4 funnel events.
     * @param string $sChannel    EventLog::CHANNEL_CAPI or EventLog::CHANNEL_PIXEL.
     * @param object $obSubject   Polymorphic subject (Phase 3.1: only Order).
     * @param string|null $sSecretKey Direct-lookup slug for /checkout/{slug}.
     * @param int $iEventTime     Meta-spec Unix seconds (paired browser+server).
     */
    public static function record(
        string $sEventId,
        string $sEventName,
        string $sChannel,
        object $obSubject,
        ?string $sSecretKey,
        int $iEventTime,
    ): bool {
        try {
            $sSubjectType = get_class($obSubject);
            $iSubjectId = self::extractSubjectId($obSubject);

            if ($iSubjectId <= 0) {
                Log::warning('Metapixel: EventLogWriter rejected subject with invalid id', [
                    'meta_pixel.subject_type' => $sSubjectType,
                    'meta_pixel.subject_id' => $iSubjectId,
                ]);
                return false;
            }

            $iSiteId = SiteResolver::getActiveSiteId();
            $sNow = (string) Carbon::now();

            // Race fence: insertOrIgnore returns affected-row count (1 = winner,
            // 0 = UNIQUE collision = loser).
            $iAffected = DB::table((new EventLog())->table)->insertOrIgnore([
                'event_id'     => $sEventId,
                'event_name'   => $sEventName,
                'channel'      => $sChannel,
                'subject_type' => $sSubjectType,
                'subject_id'   => $iSubjectId,
                'secret_key'   => $sSecretKey,
                'site_id'      => $iSiteId,
                'event_time'   => $iEventTime,
                'fired_at'     => $sNow,
                'created_at'   => $sNow,
                'updated_at'   => $sNow,
            ]);

            return $iAffected === 1;
        } catch (Throwable $obException) {
            // silent: DB-write failure must NOT cascade — Tiger-Style boundary.
            // Fail-safe: return false treats this as "lost race" → caller does
            // NOT proceed with HTTP POST / fbq call. Mirrors SendCapiEvent
            // writeFailedEvent silent-catch (T-03-22).
            Log::critical('Metapixel: EventLogWriter::record DB write FAILED', [
                'meta_pixel.event_id' => $sEventId,
                'meta_pixel.event_name' => $sEventName,
                'meta_pixel.channel' => $sChannel,
                'meta_pixel.exception' => get_class($obException),
                'meta_pixel.message' => $obException->getMessage(),
            ]);
            return false;
        }
    }

    private static function extractSubjectId(object $obSubject): int
    {
        if (!method_exists($obSubject, 'getKey')) {
            return 0;
        }
        $mKey = $obSubject->getKey();
        if (is_int($mKey)) {
            return $mKey;
        }
        if (is_string($mKey) && is_numeric($mKey)) {
            return (int) $mKey;
        }
        return 0;
    }
}
```

### Multi-site site_id Scoping

**Source:** This phase introduces it via `SiteResolver::getActiveSiteId()`.
**Apply to:** Every `EventLog::where(...)` query in production code (`OrderStatusWatcher::alreadyDispatched`, `PurchasePixel::findEventLogRow`) AND every `EventLogWriter::record` call site.

```php
$iSiteId = SiteResolver::getActiveSiteId();
$obQuery = EventLog::where(...)->where('event_name', ...)->where('channel', ...);
if ($iSiteId === null) {
    $obQuery->whereNull('site_id');
} else {
    $obQuery->where('site_id', $iSiteId);
}
```

Reason: MySQL UNIQUE treats NULL as distinct under composite keys; reads must mirror that exact semantic or single-site installs will lose visibility into their own (`site_id = NULL`) rows.

**Source:** Lovata `propertiesshopaholic/models/PropertyValueLink.php` lines 60-70 (project precedent) + `subscriptionsshopaholic/models/SubscriptionAccess.php` lines 75-77 (`$morphTo` declaration).
