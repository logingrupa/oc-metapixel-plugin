## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| migration → Lovata.OrdersShopaholic schema | Plugin tears out two columns from a third-party plugin's table. If Lovata ships a future migration adding a different column at the same position, ordering is undefined but not destructive (Laravel's `Schema::table` is column-oriented, not positional). |
| migration → MySQL UNIQUE constraint | The UNIQUE composite key is the entire race-fence for Phase 3.1. Misnaming or column-order mismatch breaks idempotency. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3.1-01 | Tampering | drop_meta_purchase_columns migration | mitigate | `down()` is fully reversible — re-adds both columns + the named index. Operator running `php artisan october:up -r {plugin}` on v1.1.0 → v1.0.3 gets the columns back. |
| T-3.1-02 | Denial of Service | drop migration on SQLite | mitigate | MIG-02 lock — `dropIndex` precedes `dropColumn` inside the closure. Laravel 12 schema-builder validates the order for SQLite. |
| T-3.1-03 | Information Disclosure | event_log table | accept | Table holds NO PII (subject_type/subject_id are FK pointers; event_id is a UUIDv4 with no entropy leakage). secret_key column mirrors what Lovata stores in `lovata_orders_shopaholic_orders.secret_key` and is not new sensitive surface. |
| T-3.1-04 | Elevation | UNIQUE constraint shape | mitigate | The 5-column UNIQUE matches the BRIEF spec letter-for-letter. PATTERNS.md call-sites in EventLogWriter::record (Wave 2) and OrderStatusWatcher::alreadyDispatched (Wave 3) read the SAME 5 columns — drift between schema and consumer code would silently weaken idempotency. Wave-4 MultiSiteEventLogTest locks the contract. |
| T-3.1-05 | Repudiation | version.yaml ledger | accept | Operators rely on `system_plugin_history` for audit. The version.yaml file is a deploy-time manifest; Phase 3.1 keeps the 1.0.1 line for historical clarity even though the file is deleted (Option A in PATTERNS.md). |
