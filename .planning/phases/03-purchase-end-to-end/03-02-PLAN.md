<objective>
Ship the eight-class custom exception hierarchy for Phase 3 (PAY-09). Every Phase 3+ production class throws a typed exception that:
- carries an immutable PHP 8.4 `readonly array $arContext` for one-liner `Log::error($sMsg, $obException->arContext)` injection,
- exposes `isRetryable(): bool` so the queue job's catch-branch logic is encoded in the type, not in a separate switch.

Purpose: Tiger-style fail-fast preconditions throw early at function boundaries (PROJECT.md "Fail-fast"). Each precondition gets a concrete grep-able class name (e.g. `OrderHasNoCurrencyException`) so log greps can pinpoint which boundary fired without parsing message strings. The transient-vs-permanent distinction in the type drives the queue-job retry-vs-dead-letter contract.

Output: One abstract base (`MetaPixelException`) + seven concrete `final` subclasses + lang keys (en/lv/ru stubs; full translations in HARD-04) + one ExceptionHierarchyTest unit test locking all the invariants.
</objective>

abstract class GoodsReceivedException extends RuntimeException
{
    public function __construct(
        string $sMessage,
        public readonly array $arContext = [],
        ?Throwable $obPrevious = null,
    ) {
        parent::__construct($sMessage, 0, $obPrevious);
    }

    protected static function jsonContext(array $arContext): string
    {
        $sJson = json_encode($arContext, JSON_UNESCAPED_SLASHES | JSON_UNESCAPED_UNICODE);
        return $sJson !== false ? $sJson : '{}';
    }
}

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| event-time exception message → log sink | Exception messages are written by trusted code (plan 03-03/03-04/03-05) but may include data from untrusted sources (order_number from XML import, currency_code from Settings). Logs sink to filesystem only. |
| `$arContext` array → log sink | Context carries internal IDs (order_id, event_id) and external metadata (graph_error, http_status). Never PII (PII is hashed by UserDataHasher before being placed in any payload). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Information Disclosure | `$arContext` log injection | mitigate | `jsonContext()` uses `JSON_UNESCAPED_SLASHES` + `JSON_UNESCAPED_UNICODE` (no extra escaping); the readonly modifier prevents downstream code from injecting attacker-controlled fields. Context keys are namespaced (`meta_pixel.*`) so any future PII addition is grep-able. |
| T-03-07 | Tampering | Subclass overriding isRetryable() to swap semantics | mitigate | All concrete classes are `final`. The base is abstract. ExceptionHierarchyTest::test_every_concrete_exception_is_final locks the invariant. A plugin consumer cannot subclass and flip retryability. |
| T-03-08 | Denial of Service | Constructing an exception with a massive $arContext array | accept | The context array is built by trusted code; the queue worker pays the serialization cost during `FailedEvent::create`. Bounded by the size of the original payload (Graph API caps payload at ~16 KB). LONGTEXT column handles it. |
| T-03-09 | Repudiation | Exception lacks attribution | accept | Exceptions are thrown by plugin code in defined call paths. The `arContext` includes `order_id` + `event_id` which are sufficient audit trail. No user identity to attribute. |
| T-03-10 | Elevation of Privilege | Lang key injection (e.g. `::lang.exception.<user-controlled>`) | mitigate | Lang keys are hard-coded constants in the throwing class — never built from runtime input. The 7 keys are an exhaustive whitelist; new keys require code edit + Phase-end review. |
