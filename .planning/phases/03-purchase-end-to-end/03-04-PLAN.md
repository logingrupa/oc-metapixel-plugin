<objective>
Ship the two transform classes that build the CAPI payload from a paid Order: `PayloadBuilder` (envelope shape + content_ids + custom_data) and `UserDataHasher` (sha256-hashed PII + plaintext request metadata + per-order CCache memoization).

Purpose: Decouple the payload construction from the HTTP boundary (03-03) and the queue dispatch (03-05). PayloadBuilder owns the Graph API envelope shape — including the content_ids contract that must byte-for-byte match the Facebook Catalog feed exporter; UserDataHasher owns the hashed-PII derivation rules + phone normalisation + guest external_id fallback. Both are stateless single-shot transforms with constructor-injectable dependencies for full test mock surface.

Output: Two production classes, one shared test fixture for real-DB Order construction, and two unit-level test suites that exhaustively assert envelope shape + every PAY-09 precondition throw + every PII normalisation rule.
</objective>
