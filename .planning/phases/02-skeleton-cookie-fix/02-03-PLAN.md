<objective>
Ship the live empty-cookie bug fix. `EnsureFbpFbcCookies` is a global Laravel HTTP middleware that sets `_fbp` (always when missing) and `_fbc` (only when `?fbclid=...` present and cookie missing) on the response, per Meta's `fb.{subdomain-index}.{creation-time}.{random}` spec. Register via Laravel's HTTP Kernel `pushMiddleware()` from `Plugin::boot()` — gated to storefront context only (skip backend + console).

Purpose: The CONTEXT establishes the live production bug: `fbevents.js` does not run early enough on SSR-cached responses / non-JS bots, so `_fbp`/`_fbc` arrive empty in CAPI envelopes and Meta cannot match users — collapsing EMQ below the v1 floor of 8. A 100%-coverage server-side middleware fixes this for every storefront response.

Output: `middleware/EnsureFbpFbcCookies.php` (Hungarian-notation Laravel middleware with the four CONTEXT Area 3 constraints baked in), an updated `Plugin::boot()` that pushes it onto the kernel stack only on storefront requests, and `tests/Feature/EnsureFbpFbcCookiesTest.php` covering all 9 invariants.

Critical pattern correction: `$this->registerMiddleware([...])` is a method that does NOT exist on October's `PluginBase` (verified at `modules/system/classes/PluginBase.php`, lines 40-291). CONTEXT and the Phase 1 audit text using that signature are wrong. This plan uses the Laravel-native `$this->app->make(\Illuminate\Contracts\Http\Kernel::class)->pushMiddleware(...)` per PATTERNS.md lines 146-164.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser query string (`?fbclid=...`) -> response cookie `_fbc` | Untrusted user-controlled input flows directly into a cookie value. Length/charset validation required to prevent injection. |
| `Request::getHost()` -> subdomain-index | Host header is client-controlled; could be spoofed to game subdomain-index. Capping at 2 + relying on Laravel's `TrustedHosts` for production limits exposure. |
| Middleware execution scope | Runs on every storefront response, including endpoints that may not currently set cookies. Threat = unexpected Set-Cookie header on cacheable responses (CDN cache poisoning). |
