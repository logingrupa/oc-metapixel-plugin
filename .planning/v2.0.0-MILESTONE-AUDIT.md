## Cross-Phase Integration

End-to-end runtime flows all WIRED:

1. **Purchase flow** (Phase 3 → Phase 2 → Phase 4 → Phase 3 browser side) — Order saved → OrderStatusWatcher → SendCapiEvent::dispatch → AdapterRegistry::resolveByClass → Settings::lookupForSite → MetaClient::sendForPixel (v23.0) → EventLogWriter race-fence → EventPixel browser render with same event_id. WIRED.
2. **AddToCart flow** — CartPosition::created → CartPositionWatcher → SendCapiEvent dispatch with `shopaholic.cart_position` alias. WIRED.
3. **Theme event flow** — Twig `this.metapixel.pushEvent` → ThemeEventCollector → PixelHead emits fbq blocks → optional `also_dispatch_capi:true` mirrors via SendCapiEvent. Larajax `Metapixel::onFireEvent` → ThemeAjaxHandler (META_STANDARD allowlist + Settings::getThemeCustomEventNames). WIRED.
4. **Cookie middleware flow** — Inbound → EnsureFbpFbcCookies → kill switch + HostIndexResolver subdomain index + CR-03 fbclid validation + fb.{N}.{ms}.{16-hex} format. WIRED.
5. **FailedEvents backend flow** — Settings menu → ListController → batch onReplay/onCheckDedup → MetaClient. WIRED (batch only).
6. **Multisite routing** — Site-A Order → pixel_A; Site-B Order → pixel_B. `$propagatable = []` + DB-bypass lookup + EventLog UNIQUE NULL-distinct semantics. WIRED.

**Defects:**

- **BLOCKER — CI minimal-install cell** (TOOL-09 partial): `.github/workflows/metapixel-qa.yml:99` `--exclude-testsuite='Metapixel Adapter Tests'` is a silent no-op (testsuite removed Phase 3 in favour of `#[Group('adapter')]` per CLAUDE.md). Fix: replace with `--exclude-group=adapter`.
- **WARNING — composer-dependency-analyser scope**: Plugin.php top-level `use Lovata\OrdersShopaholic\Models\Order|CartPosition` outside the path allowlist (`classes/adapter/shopaholic`, `classes/event/adapter/shopaholic`). Minimal install would flag DEV_DEPENDENCY_IN_PROD. Runtime safe (compile-time alias, never autoloaded under isShopaholicEnabled gate). Fix: add `Plugin.php` to path allowlist, OR fully-qualify the two class references at use-site instead of importing.
- **WARNING — Single-row FailedEvents UI absent**: Controller exposes `onReplay(int)` / `onCheckDedup(int)` but `_list_toolbar.htm` / `config_list.yaml` only wire batch buttons. Per-row data-request attribute missing from list views. Phase 4 VERIFICATION listed batch-only as design intent (D-07); single-row handlers are dead reachability without an inline action button.
